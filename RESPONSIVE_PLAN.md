# Фаза 7 — Адаптивность под мобильные/планшетные экраны: план

`ROADMAP.md`, Фаза 7: «текущая вёрстка использует Tailwind-брейкпоинты частично, но не проверена целиком на реальных мобильных/планшетных экранах» — нужно пройтись по каждой странице на 375px/768px/1024px: переполнение по горизонтали, читаемость форм в колонку, размер тач-таргетов, работа `RouteMap`/pulse-маркеров.

## Текущее состояние (проверено чтением кода — не браузером)

Брейкпоинты `sm:`/`md:`/`lg:` уже расставлены довольно широко (не голая невёрстка) — работа здесь **аудит существующего + точечные фиксы**, не вёрстка с нуля.

- **`components/ui/Header.tsx`** — есть мобильное меню (`useState menuOpen`, паттерн `md:hidden`/`hidden md:flex`), закрывается сам при смене роута (`prevPathname`-эффект). Не проверено: реальный размер тач-таргетов пунктов мобильного меню и кнопки-гамбургера (Apple/Google рекомендация — минимум 44×44px), поведение при очень длинном имени пользователя (`hidden lg:flex` блок с именем — что если оно длинное на границе 1024px).
- **`components/routes/RouteMap.tsx`** — декоративная карта (см. отдельный незакрытый пункт ROADMAP про замену на Leaflet, это не она). Маркеры (`MARKERS`) — `position: absolute`, `top`/`left` в `%`, подписи городов — `whitespace-nowrap` с абсолютным позиционированием `top-4 left-1/2 -translate-x-1/2`. **Риск:** на узком контейнере (375px) подпись у маркера с `left: 60%` (Chișinău) или `left: 42%` может вылезать за правый/левый край контейнера — `whitespace-nowrap` не даёт тексту переноситься, а контейнер `overflow-hidden` — значит подпись будет **обрезана**, не перенесена. Нужно проверить визуально на 375px, не только прочитать код.
- **`components/home/SearchWidget.tsx`** — `grid-cols-1` → `md:grid-cols-2` → `lg:grid-cols-5`. Значит на планшете (768px, `md`, но ещё не `lg`) форма — 2 колонки на 5 полей (неровное распределение, последнее поле повиснет одно в ряду). Не обязательно баг, но визуально стоит проверить, не выглядит ли это сломанным.
- **`components/routes/ActiveRoutesSidebar.tsx`** — `w-full` → `md:w-80` (жёсткая ширина 320px с `md`). В связке с `app/routes/page.tsx` (`flex-col md:flex-row`) — на `md` (768px) сайдбар фиксированной ширины 320px + основной контент в той же строке — на границе 768px может быть тесно, зависит от контента карточек маршрутов рядом.
- **Все проверенные страницы** (`app/page.tsx`, `app/routes/page.tsx`, `app/about/page.tsx`) используют `max-w-[1280px] px-4 sm:px-12` — паттерн консистентный, горизонтальных переполнений на уровне контейнера не ожидается, но дочерние абсолютно позиционированные элементы (как `RouteMap`) могут вылезать независимо от родителя.
- **Не проверены вообще** (не грепались точечно) — `app/trips/page.tsx`, `app/trips/[tripId]/seats/page.tsx` (выбор мест — вероятно самая рискованная страница на мобильном, обычно грид мест плохо ужимается), `app/checkout/[orderId]/page.tsx`, `app/bookings/page.tsx`, `app/login/page.tsx`/`app/signup/page.tsx`.

## Как проверять (инструмент)

Chrome/Firefox DevTools → Device Toolbar (`Cmd+Shift+M` в Chrome) → выставить кастомные размеры **375px** (мобильный, напр. iPhone SE/12 mini), **768px** (планшет, iPad portrait), **1024px** (iPad landscape / маленький ноутбук) — это и есть три контрольные точки из формулировки задачи в `ROADMAP.md`. Для каждой страницы на каждой ширине смотреть:

1. Горизонтальный скролл на `<body>` — если появляется, что-то вылезает за пределы вьюпорта (частая причина: фиксированная ширина/`whitespace-nowrap`/абсолютное позиционирование без учёта краёв).
2. Реальная читаемость текста и кнопок — не заезжают ли элементы друг на друга.
3. Тач-таргеты (кнопки/ссылки/пункты меню) — на глаз не мельче ~44×44px, актуально в первую очередь для мобильного меню `Header`.
4. Формы (`SearchWidget`, login/signup, checkout) — поля не должны сжиматься до нечитаемой ширины, лейблы не должны переноситься криво.

## Найденная и исправленная проблема — ✅ СДЕЛАНО

**Тач-таргеты мобильного меню `Header.tsx` были меньше рекомендованного минимума 44×44px** (Apple HIG/Material Design):
- Кнопка-гамбургер (`aria-label="Toggle menu"`) — фактический кликабельный размер **24×30px** (замерено через `getBoundingClientRect()` в Playwright).
- Пункты раскрытого мобильного меню (About Us/Schedules/Tickets/My bookings/Sign out) — высота **36px** (`py-2` при `text-sm`).

**Исправлено:**
- Кнопка-гамбургер — `className="-mr-2.5 flex h-11 w-11 items-center justify-center md:hidden"` (44×44px, `-mr-2.5` компенсирует добавленный паддинг, чтобы иконка не сдвигала общую вёрстку хедера вправо).
- Все пункты мобильного меню — добавлен `flex min-h-11 items-center` (44px минимальная высота, контент внутри вертикально центрирован).

Проверено вживую (до и после фикса, через Playwright `getBoundingClientRect()` + визуальный скриншот): до фикса — 24×30 / 36px высоты, после — 44×44 / 44px высоты на всех пунктах. Скриншот открытого меню подтверждает, что вёрстка не сломалась (нет наложений, интервалы между пунктами не выглядят чрезмерными). `npx tsc --noEmit` чистый; `npm run lint` показывает 1 pre-existing ошибку в `auth/AuthProvider.tsx` (`react-hooks/set-state-in-effect`), не связанную с этим изменением и не в `Header.tsx` — подтверждено через `git stash`/повторный lint на состоянии до этого изменения, ошибка была там же.
